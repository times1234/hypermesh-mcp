# 迁移到另一台电脑步骤

本文只说明把当前 HyperMesh 自动划分脚本迁移到另一台电脑后要做什么。

## 1. 复制整个项目文件夹

把当前整个文件夹复制到新电脑，例如仍然放到：

```text
E:\mcp\hypermesh-mcp-server
```

也可以放到其他路径，例如：

```text
D:\tools\hypermesh-mcp-server
```

如果换了路径，后面命令里的路径也要一起改成新电脑上的真实路径。

## 2. 确认新电脑已经安装 HyperMesh

新电脑必须已经安装并能正常打开 Altair HyperMesh。

这个脚本不是独立网格引擎，实际划分仍然由 HyperMesh 执行。

## 3. 确认新电脑已经安装 Python

推荐安装：

```text
Python 3.12 64-bit
```

安装后打开 PowerShell，检查：

```powershell
python --version
```

如果能看到 Python 版本号，说明 Python 可以使用。

## 4. 安装 Python 依赖

如果项目文件夹里带有 `wheels` 文件夹和 `install_offline_dependencies.bat`，直接双击：

```text
install_offline_dependencies.bat
```

这个 bat 会自动在当前项目目录里安装离线依赖。

如果不想双击，也可以在 PowerShell 中运行：

```powershell
cd E:\mcp\hypermesh-mcp-server
python -m pip install --no-index --find-links wheels -r requirements.txt
```

如果项目放在其他路径，把 `cd` 后面的路径改掉，例如：

```powershell
cd D:\tools\hypermesh-mcp-server
python -m pip install --no-index --find-links wheels -r requirements.txt
```

安装完成后检查：

```powershell
python -c "import mcp; print('mcp OK')"
```

如果输出：

```text
mcp OK
```

说明依赖安装成功。

## 5. 在 HyperMesh 中打开自动划分面板

打开 HyperMesh，然后在 HyperMesh 的 Tcl 命令窗口中运行：

```tcl
source "E:/mcp/hypermesh-mcp-server/launch_meshing_workflow_panel.tcl"
```

如果项目放在其他路径，要改成新电脑上的真实路径，例如：

```tcl
source "D:/tools/hypermesh-mcp-server/launch_meshing_workflow_panel.tcl"
```

运行后会出现 HyperMesh 自动网格划分面板。

## 6. 开始划分

在面板中选择或确认参数后，点击：

```text
开始划分
```

如果要使用后台划分功能，点击：

```text
后台划分
```

后台划分需要新电脑能找到 `hmbatch.exe`。如果 HyperMesh 安装路径和原电脑不同，并且后台划分启动失败，需要检查代码或配置中使用的 HyperMesh 批处理路径。

## 7. 输出文件位置

运行过程中会自动生成：

```text
runs\
outputs\
```

常见输出：

- `outputs\*.hm`：最终模型
- `runs\workflow_report_*.txt`：中文报告
- `runs\workflow_probe_*.txt`：探针结果
- `runs\workflow_diagnostics_*.jsonl`：诊断日志

`runs` 和 `outputs` 不需要提前创建，脚本运行时会自动生成。

## 8. 常见检查

如果面板打不开，先检查：

```text
launch_meshing_workflow_panel.tcl 路径是否写对
```

如果 Python 依赖报错，重新运行：

```text
install_offline_dependencies.bat
```

如果后台划分启动失败，检查：

```text
HyperMesh 是否安装
hmbatch.exe 路径是否存在
Python 是否能正常运行
```

如果普通前台划分能用，但后台划分不能用，优先检查 `hmbatch.exe` 路径。

## 9. 一键配置后台划分 hmbatch 路径

如果后台划分日志里提示：

```text
Could not find hmbatch.exe
```

在新电脑项目目录里双击：

```text
configure_hmbatch_path.bat
```

这个脚本会自动查找 `hmbatch.exe`，找到后会做两件事：

1. 在项目目录写入：

   ```text
   hypermesh_batch_path.txt
   ```

2. 设置用户环境变量：

   ```text
   HYPERMESH_BATCH_EXE
   ```

双击运行成功后，在 HyperMesh 中重新 source 面板：

```tcl
source "D:/qcy/hypermesh-mcp-server/launch_meshing_workflow_panel.tcl"
```

路径要改成新电脑上的真实路径。

然后重新点击：

```text
后台划分
```

如果自动查找失败，就手动找到 `hmbatch.exe`，然后把完整路径写入：

```text
hypermesh_batch_path.txt
```

例如文件内容可以是：

```text
D:\Program Files\Altair\2020\hwdesktop\hm\bin\win64\hmbatch.exe
```

如果后台划分一点击就提示：

```text
invalid command name "puts"
```

说明新电脑正在使用旧版 `launch_meshing_workflow_panel.tcl`，或者新电脑的 HyperMesh Tcl 环境里没有标准 `puts` 命令。

处理方法：

1. 从当前电脑复制最新的：

   ```text
   launch_meshing_workflow_panel.tcl
   ```

2. 覆盖到新电脑项目目录中的同名文件。

3. 在新电脑 HyperMesh 中重新 source 面板：

   ```tcl
   source "D:/tools/hypermesh-mcp-server/launch_meshing_workflow_panel.tcl"
   ```

   路径要改成新电脑上的真实路径。

4. 重新点击：

   ```text
   后台划分
   ```
